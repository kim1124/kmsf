# GridStack Resize Boundary Exit Recovery Plan

## Goal

위젯 resize 중 포인터가 GridLayout 영역 또는 브라우저 viewport 경계를 벗어난 뒤 interaction 종료가 누락되는 문제를 E2E로 재현하고, `@kmsf/gridstack` adapter 경계에서 안전하게 종료 처리하는 최소 수정 계획을 세운다.

## Current Evidence

- `gridstack` installed version: `12.6.0`.
- GridStack 공식 lifecycle은 `resizestart`, `resize`, `resizestop`이며, `resizestop`은 resize 종료 후 DOM attribute가 갱신된 뒤 발생한다.
- `src/gridstack/adapter.ts`는 `dragstart`/`resizestart`에서 `isInteracting = true`로 전환하고, `dragstop`/`resizestop`에서만 `isInteracting = false` flush를 예약한다.
- GridStack `DDResizableHandle`은 resize 시작 후 `document`에 `mousemove`/`mouseup` listener를 붙인다. 따라서 같은 document 안에서의 `mouseup`은 처리될 수 있지만, 브라우저 밖 release처럼 document에 `mouseup`이 전달되지 않는 경로에서는 `resizestop` 자체가 누락될 수 있다.
- 기존 E2E `defers grid sync while a widget is actively resizing`는 정상 `page.mouse.up()` 경로만 검증한다. 2026-06-08 focused run은 통과했다.

## Root Cause Hypothesis

현재 adapter는 GridStack의 stop event를 유일한 interaction 종료 신호로 신뢰한다. 브라우저 이탈, window blur, document visibility change 등으로 GridStack 내부 `mouseup` handler가 실행되지 않으면 `resizestop`이 발생하지 않고, adapter의 `pendingSync`/`pendingCommit` flush도 실행되지 않는다. 그 결과 resize helper class 또는 interaction state가 남고, 이후 layout sync나 다음 interaction이 막힐 수 있다.

## Approach Options

### Option A: Adapter-level boundary exit guard

`createDashboardGridAdapter()` 내부에서 active drag/resize 동안만 document/window guard listener를 임시 등록한다. `documentElement.mouseleave`, `window.blur`, `visibilitychange=hidden` 같은 browser-boundary 신호가 들어오면 마지막 pointer 위치로 synthetic `mouseup`을 document에 dispatch해 GridStack의 기존 cleanup path를 먼저 실행하게 한다. 그래도 stop event가 발생하지 않으면 adapter state만 fallback flush한다.

Recommendation: 이 방식을 우선한다. public API 변경이 없고, GridStack 내부 구현을 직접 patch하지 않으며, 현재 adapter의 interaction defer 구조와 가장 작게 맞물린다.

### Option B: GridStack dependency patch or fork

GridStack `DDResizableHandle`/`DDDraggable` 내부에 viewport exit guard를 추가한다.

Not recommended for this package change. dependency patch는 유지보수 비용이 크고, `gridstack: latest` 정책과 충돌한다.

### Option C: E2E만 추가하고 runtime 보강 보류

그리드 밖 `mouseup` 정상 경로만 회귀 테스트로 추가한다.

Insufficient. 사용자가 제기한 핵심은 종료 event 누락 상태이며, 브라우저 밖 release는 정상 `mouseup` 경로로는 재현되지 않는다.

## E2E Plan

### Case 1: Grid area exit with document mouseup

Purpose: 포인터가 GridLayout 밖으로 이동해도 browser document 안에서 `mouseup`이 전달되면 interaction이 정상 종료되는지 확인한다.

File: `test/playwright/specs/dashboard-grid.spec.ts`

Add helper:

```ts
async function readWidgetInteractionState(widget: Locator) {
  return widget.evaluate((element) => ({
    isResizing: element.classList.contains("ui-resizable-resizing"),
    hasInlinePosition: (element as HTMLElement).style.position === "absolute",
  }));
}
```

Test outline:

```ts
test("finishes widget resize after leaving the grid area", async ({ page }, testInfo) => {
  test.skip(testInfo.project.name !== "chromium", "Pointer interaction regression runs on the desktop project only.");

  const diagnostics = collectBrowserDiagnostics(page);

  await page.goto("/");

  const grid = page.getByTestId("dashboard-grid");
  const sales = page.getByTestId("dashboard-widget-sales");
  const beforeResize = await readWidgetLayout(sales);
  const { startX, startY } = await startWidgetResize(page, sales);
  const gridBox = await grid.boundingBox();

  if (!gridBox) {
    throw new Error("Grid bounding box is not available");
  }

  await page.mouse.move(startX + 180, startY + 120, { steps: 8 });
  await page.mouse.move(gridBox.x + gridBox.width + 80, startY + 120, { steps: 8 });
  await page.mouse.up();

  await expect.poll(async () => (await readWidgetInteractionState(sales)).isResizing).toBe(false);
  await expect.poll(async () => {
    const layout = await readWidgetLayout(sales);

```

Expected RED:

- 이 테스트가 현재 통과하면 정상 document-level mouseup 경로는 이미 GridStack이 처리하는 것으로 기록한다.
- 실패하면 browser-exit guard 이전에 GridStack document mouseup 경로부터 원인을 재분석한다.

### Case 2: Browser boundary exit without document mouseup

Purpose: document `mouseup`이 누락되는 browser-boundary 상황에서 adapter가 interaction을 종료하고 pending sync를 flush하는지 확인한다.

Add helper:

```ts
async function simulateBrowserBoundaryExit(page: Page) {
  const viewport = page.viewportSize() ?? { width: 1280, height: 720 };

  await page.mouse.move(viewport.width - 1, viewport.height - 1, { steps: 8 });
  await page.evaluate(() => {
    document.documentElement.dispatchEvent(
      new MouseEvent("mouseleave", {
        bubbles: true,
        cancelable: true,
        buttons: 1,
        clientX: window.innerWidth + 24,
        clientY: window.innerHeight + 24,
        relatedTarget: null,
      }),
    );
    window.dispatchEvent(new Event("blur"));
  });
}
```

Test outline:

```ts
test("finalizes widget resize when the pointer leaves the browser boundary", async ({ page }, testInfo) => {
  test.skip(testInfo.project.name !== "chromium", "Pointer interaction regression runs on the desktop project only.");

  const diagnostics = collectBrowserDiagnostics(page);

  await page.goto("/");

  const grid = page.getByTestId("dashboard-grid");
  const sales = page.getByTestId("dashboard-widget-sales");

  await expect(grid).toHaveAttribute("data-columns", "6");

  try {
    const { startX, startY } = await startWidgetResize(page, sales);
    await page.mouse.move(startX + 140, startY + 110, { steps: 8 });

    await page.getByLabel("컬럼 선택").evaluate((element) => {
      const select = element as HTMLSelectElement;
      select.value = "12";
      select.dispatchEvent(new Event("change", { bubbles: true }));
    });

    await expect.poll(() => readGridEngineColumn(grid)).toBe(6);

    await simulateBrowserBoundaryExit(page);

    await expect.poll(async () => (await readWidgetInteractionState(sales)).isResizing).toBe(false);
    await expect(grid).toHaveAttribute("data-columns", "12");
    await expect.poll(() => readGridEngineColumn(grid)).toBe(12);
    expect(diagnostics).toEqual([]);
  } finally {
    await page.mouse.up().catch(() => undefined);
  }
});
```

Expected RED:

- 현재 adapter에는 `mouseleave`/`blur` guard가 없으므로 `isResizing` 또는 engine column assertion이 실패해야 한다.
- RED가 다른 이유로 실패하면 production code 변경 전에 실패 원인을 보고한다.

### Case 3: Follow-up interaction after forced end

Purpose: forced end 이후 다음 resize가 다시 정상 시작/종료되는지 확인한다.

Scope decision: Case 2에서 다음 `resizeWidget(page, sales, 80, 60)`까지 이어붙이면 테스트 시간이 적고 회귀 가치가 높다. 다만 RED 원인을 분리하기 어렵다면 Case 2 green 이후 별도 테스트로 분리한다.

## Runtime Implementation Plan

Update 2026-06-08 11:17 KST: the blur-immediate forced-end approach below was superseded. Final implementation keeps resize active while the mouse button remains pressed and finalizes only when a later mouse event reports `buttons === 0`; DD binding recovery uses `grid.prepareDragDrop(item, true)`.

File: `src/gridstack/adapter.ts`

1. Track the latest pointer coordinates only while `isInteracting` is true.
2. Attach guard listeners on `startInteraction()`:
   - `document.addEventListener("mousemove", handleDocumentMouseMove, true)`
   - `document.documentElement.addEventListener("mouseleave", handleDocumentMouseLeave, true)`
   - `document.addEventListener("visibilitychange", handleVisibilityChange)`
3. Detach the same listeners on `stopInteraction()`, `flushInteraction()`, and `destroy()`.
4. On browser-boundary signal, dispatch one synthetic `MouseEvent("mouseup")` to `document` using the last pointer coordinates. This should let GridStack run its own private cleanup and emit `resizestop`/`dragstop`.
5. If `resizestop`/`dragstop` still does not arrive by the next animation frame, call the adapter's existing `stopInteraction()` as a fallback so React-facing pending commit/sync is not permanently blocked.
6. Keep all new state inside `createDashboardGridAdapter()`; do not expose new public props or raw GridStack internals.

Implementation sketch:

```ts
type PointerSnapshot = {
  clientX: number;
  clientY: number;
};

let lastPointer: PointerSnapshot | undefined;
let forceEndFrame: number | undefined;

const captureInteractionPointer = (event: MouseEvent) => {
  lastPointer = { clientX: event.clientX, clientY: event.clientY };
};

const forceInteractionEnd = () => {
  if (!isInteracting) {
    return;
  }

  cancelFrame(forceEndFrame);
  forceEndFrame = window.requestAnimationFrame(() => {
    forceEndFrame = undefined;
    const point = lastPointer ?? { clientX: 0, clientY: 0 };

    document.dispatchEvent(
      new MouseEvent("mouseup", {
        bubbles: true,
        cancelable: true,
        button: 0,
        buttons: 0,
        clientX: point.clientX,
        clientY: point.clientY,
      }),
    );

    if (isInteracting) {
      stopInteraction();
    }
  });
};
```

Guard detail:

- `mouseleave` should only force end when `event.relatedTarget === null`, because child-element transitions are not browser exits.
- `visibilitychange` should only force end when `document.visibilityState === "hidden"`.
- If real `mouseup` arrives first, existing `resizestop`/`dragstop` path should detach guards and cancel fallback frames.
- No GridStack private method should be called directly.

## Residual Risks

- Playwright can simulate `mouseleave`/`blur`, but it cannot perfectly reproduce every OS-level browser-window release path. If the automated E2E passes, a short headed/manual browser check is still useful before closing implementation.
- Synthetic `mouseup` is intentionally scoped to active GridStack interactions. If fired too broadly, it could prematurely end legitimate long drags. Guard attachment must therefore be active only between `dragstart`/`resizestart` and stop/cleanup.
- Touch/pointer interactions on mobile are not the first target. Desktop Chromium should be fixed first; mobile pointer coverage can be added if the same symptom is confirmed on touch devices.
