# Product Goals

## 목적

이 템플릿은 Next.js 기반 프로젝트를 빠르게 시작하기 위한 공용 베이스다. 목표는 다음과 같다.

- 신규 프로젝트를 `복사` 또는 `Fork`하여 즉시 개발을 시작할 수 있어야 한다.
- 인증, 권한, 다국어, 상태 관리, 공통 레이아웃, DB 연동, 테스트 체계를 처음부터 일관된 방식으로 제공해야 한다.
- 팀 내 개발자들이 프로젝트마다 기본 구조를 다시 고민하지 않도록 해야 한다.
- 서버 사이드 렌더링 중심 구조를 유지하면서, 필요한 영역만 클라이언트 하이드레이션으로 확장할 수 있어야 한다.

## 템플릿의 범위

이번 단계에서 정의할 범위:

- Next.js App Router 기반 구조
- TypeScript 기본 설정
- Tailwind CSS + shadcn/ui 기반 UI 시스템
- 다국어 구조
- 인증/보안 구조
- Supabase 연동 구조
- 테스트 전략
- 모바일/데스크톱 공통 레이아웃 전략
- 향후 플러그인 확장을 위한 구조 설계

이번 단계에서 구현하지 않고 설계만 정리할 범위:

- `echarts` 플러그인 프로젝트
- 데이터 테이블 플러그인 프로젝트
- 실제 운영용 멀티 테넌트 정책
- 권한(Role/Permission) 세부 정책

## 성공 기준

- 개발자가 템플릿을 생성한 뒤 최소한의 환경변수 설정만으로 로컬 실행이 가능하다.
- 인증, 다국어, 공통 레이아웃, 데이터 접근 방식이 문서만 보고도 이해된다.
- SSR 우선 원칙과 클라이언트 컴포넌트 사용 기준이 분명하다.
- 운영 환경으로 확장할 때 구조 변경이 최소화된다.

## 템플릿 기본 원칙

- 라우팅은 Next.js App Router를 기준으로 한다.
- 기본 렌더링은 서버 컴포넌트 우선으로 한다.
- 클라이언트 컴포넌트는 사용자 상호작용, 브라우저 API, 실시간 상태가 필요한 경우로 제한한다.
- 공통 기능은 재사용 가능한 모듈로 분리한다.
- 플러그인 형태의 확장은 메인 템플릿을 오염시키지 않도록 느슨하게 결합한다.
