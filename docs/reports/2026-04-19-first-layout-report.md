# First Layout Project Report

작성일:
- 2026-04-19

## 적용된 지침

- i18n 기본 locale은 `ko`
- 초기 권한 모델은 `admin`, `member`
- MFA는 선택 기능으로 설계
- 개발 환경 및 Vercel 배포 환경에서 Supabase 인증을 기본 적용할 수 있도록 구조 반영
- 화이트 기반 Light 테마에 민트/라임 포인트 컬러 적용

## 이번 작업 결과

- Next.js App Router 기반 프로젝트 생성
- `next-intl` 기반 locale 구조 추가
- `Supabase Auth` 서버/브라우저 클라이언트 유틸 추가
- 이메일/비밀번호 로그인 및 Google OAuth 진입 액션 추가
- `manager.username -> email` 매핑 기반 ID 로그인 구조 반영
- 계정이 없을 때 진입하는 초기 관리자 설정 페이지 구조 추가
- 프로필 팝업 내 계정 수정 및 회원 탈퇴 흐름 추가
- 보호 레이아웃, 좌측 사이드바, 상단 헤더, 모바일 하단 네비게이션 추가
- `admin/member` 권한 타입 추가
- shadcn/ui 호환 구조(`components.json`, `ui` 컴포넌트) 반영
- `.env.example`, `.env.development`, `.env.production` 초안 추가
- Vitest 기본 설정과 역할 정규화 테스트 추가

## 사용자 피드백 반영 메모

- URL에 `ko`를 강제하지 않고 `<html lang>` 중심으로 처리하는 방향으로 수정
- 로그인 화면은 `KMSF 로그인` 타이틀만 남기고 부가 설명 제거
- 로그인/회원가입 폼의 라벨과 입력 필드 간 세로 간격을 `5px`로 통일
- Google 로그인 버튼 앞에 Google 마크 아이콘 추가
- 라이트 테마는 화이트 배경과 민트 포인트를 기본값으로 유지하도록 보정
- 다크 테마는 선택형으로 유지하고, 이전 선택은 쿠키로 저장
- 차트/데이터/설정 페이지 404 문제를 해결하도록 상위 라우트를 보강
- 회원가입 실패 원인을 숨기지 않도록 Supabase 에러 로그와 UI 메시지 분기 보강

- 실제 Supabase 프로젝트를 연결할지, 로컬 Supabase CLI 스택부터 붙일지
- `member`라는 명칭을 그대로 유지할지, 요청하신 표현대로 `일반인` 또는 `general`로 바꿀지
- MFA를 선택 기능으로 둘 때, 관리자 계정만 우선 권장 정책으로 둘지
- 로그인 외에 회원가입, 비밀번호 재설정, 계정 연결 페이지를 1차 범위에 포함할지
