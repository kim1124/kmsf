KMSF가 뭔가요?

한 줄 요약

KMSF는 "Next.js 관리자 대시보드 만드는 회사 표준 보일러플레이트"이고, create-kmsf는 그 보일러플레이트를 새 프로젝트로 복사해주는 명령어 도구다.

1.1 단어부터 정리하기

KMSF 저장소 안에서 자주 보게 될 단어들입니다. 일단 외우려 하지 말고 "아 이런 게 있구나" 정도로 넘기세요.

단어일상어 비유실제 의미
Next.js"웹사이트 만드는 레고 박스"React 기반 웹 프레임워크
Admin dashboard"회사 내부 관리자 페이지"로그인 → 데이터 조회 → 설정 변경
모노레포"여러 프로젝트가 한 폴더에"한 git 저장소에 여러 패키지가 함께 있는 구조
Boilerplate"붕어빵 틀"새 프로젝트 시작할 때 매번 똑같이 쓰는 코드
Scaffold"건물 짓기 전 비계"새 프로젝트의 기본 뼈대를 자동으로 만들어주는 일
CLI"검은 화면에 명령어로 동작하는 도구"Command Line Interface = 터미널에서 쓰는 프로그램
npx"인터넷에서 한 번 받아서 바로 실행"npm이 제공하는 임시 실행 도구

1.2 KMSF의 정체

KMSF는 그냥 "회사 안에 굴러다니는 Next.js 관리자 페이지 표준"이라고 보면 됩니다. 다음 두 가지 역할을 동시에 합니다.

① 실제로 동작하는 reference 앱

apps/kmsf/에 들어있는 진짜 동작하는 admin dashboard입니다. 로그인, 다국어, 어두운 모드, 데이터 테이블 등이 다 들어있습니다.

② 새 프로젝트의 출발점 (template)

위 reference 앱을 자동으로 새 폴더에 복사해주는 도구가 npx create-kmsf입니다. 신입 개발자가 매번 처음부터 만들 필요가 없습니다.

1.3 그래서 create-kmsf가 뭐냐면…

커피 머신을 생각해 봅시다.

사용자가 명령 한 줄을 치면, 자동으로 새 프로젝트 폴더가 완성된다.

1.4 왜 만들었나?

매번 새 admin dashboard를 만들 때마다 다음을 반복하면 비효율적입니다.

  1. Next.js 설치 → React 19 + TypeScript 설정
  2. Tailwind CSS + Radix UI 추가
  3. 로그인 시스템 만들기 (Supabase or local-json)
  4. 다국어(ko/en) 라우팅 설정
  5. 다크 모드 토글
  6. e2e 테스트 환경 (Playwright) 설정
  7. … (보통 1주일 이상 걸림)

이 모든 걸 미리 해놓은 게 KMSF이고, 한 번에 복사할 수 있게 만든 게 create-kmsf입니다.

💡 신입에게 중요한 포인트

지금 당장 코드를 100% 이해할 필요 없습니다. 우선 "create-kmsf는 새 프로젝트 만들어주는 도구" 정도만 알면 충분합니다. 나머지는 차근차근 배웁니다.

1.5 한 번 직접 실행해볼래요?

아래 명령을 그대로 복사해서 터미널에 붙여넣어 보세요. 임시 폴더에 새 프로젝트가 만들어집니다.

터미널 
cd /tmp
mkdir kmsf-test
cd kmsf-test
node /Users/kim1124/개발/kmsf/packages/create-kmsf/bin/create-kmsf.js my-first-app \
  --auth=local-json --no-i18n --no-install --no-git --no-playwright --silent
ls my-first-app/

잘 됐다면 my-first-app/ 폴더 안에 package.json, src/, .env.local 등이 보입니다. 그게 새 프로젝트입니다.
