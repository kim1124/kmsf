Chapter 3 / 10

npx 명령을 입력하면?

한 줄 요약

npx create-kmsf my-app은 → bin/create-kmsf.js를 실행 → 그 안에서 dist/cli.js의 runCli()를 호출 → 결과 exit code로 종료. 3단계뿐이다.

3.1 사용자가 입력하는 명령

터미널 
npx create-kmsf my-app --auth=local-json

이 한 줄을 컴퓨터가 어떻게 풀어 읽는지 천천히 따라가 봅시다.

3.2 스텝 1: npx가 무엇을 하나

npx는 npm이 제공하는 "한 번 실행하는 도구"입니다. 동작은 이렇습니다.

  1. 로컬에 create-kmsf가 설치되어 있는지 확인
  2. 없으면 npm registry(인터넷)에서 임시로 다운로드
  3. 다운로드한 패키지의 package.json → "bin" 필드를 본다
  4. bin이 가리키는 파일을 Node.js로 실행

📦 우리 저장소에서는 어떻게?

지금은 사내 모노레포라 npm registry에 publish된 게 아닙니다. 그래서 직접 node packages/create-kmsf/bin/create-kmsf.js로 실행합니다. 동작은 npx와 동일합니다.

3.3 스텝 2: bin/create-kmsf.js 의 역할

이 파일을 한 번 들여다 봅시다. 정말 짧습니다.

packages/create-kmsf/bin/create-kmsf.js 
#!/usr/bin/env node
// dist/ 의 컴파일된 CLI를 로드한다. npm run build를 먼저 실행해야 함.
import { runCli } from "../dist/cli.js";

runCli().then(
  (code) => process.exit(code),
  (e) => {
    process.stderr.write(`Fatal: ${e?.message ?? e}\n`);
    process.exit(99);
  },
);

이게 전부입니다. 즉 bin 파일은 단지 "진짜 CLI 코드를 호출해서 exit code로 종료"하는 얇은 래퍼(wrapper)입니다. 비유하자면 사무실 입구의 안내데스크 같습니다 — 손님(사용자)을 받아서 담당 부서(runCli)로 안내만 합니다.

3.4 스텝 3: 실제 코드는 dist/cli.js에서

bin은 ../dist/cli.js에서 runCli를 import합니다. 이 dist는 어디서 왔을까요?

TypeScript 원본은 직접 실행 못 하니까 한 번 변환한 후 실행한다.

⚠️ Node.js는 .ts를 직접 못 읽어요

그래서 tsc (TypeScript 컴파일러)로 한 번 .js로 변환하고, 이걸 dist/ 폴더에 둡니다. 사용자가 node bin/create-kmsf.js를 실행하면 ../dist/cli.js를 읽기 때문에 빌드를 먼저 해야 합니다.

3.5 전체 호출 계통도

여기까지를 한눈에 보면:

사용자 명령
npx create-kmsf my-app --auth=local-json
npx
패키지 다운로드/조회 → bin 필드 확인
bin/create-kmsf.js
../dist/cli.js 에서 runCli() import
dist/cli.js (= src/cli.ts 컴파일 결과)
runCli(process.argv.slice(2)) 호출
runCli() 내부
→ 이 부분이 다음 챕터의 주제
process.exit(code)
0 = 성공, 1~99 = 다양한 실패

3.6 generator-core는 어디 끼어들까?

위 흐름에서 runCli()가 일을 하다가 scaffold 관련 작업이 필요해지면 @kmsf/generator-core를 import해서 호출합니다. 즉 generator-core는 cli.ts에 의해 사용되는 도구 상자입니다.

src/cli.ts (발췌) 
import {
  scaffold,
  detectPackageManager,
  ScaffoldError,
  TargetExistsError,
} from "@kmsf/generator-core";  // ← generator-core 라이브러리 import

그래서 generator-core도 마찬가지로 dist/가 빌드되어 있어야 합니다.

3.7 직접 확인해보기

터미널 
# 1) 두 패키지 모두 빌드
npm --workspace=packages/generator-core run build
npm --workspace=packages/create-kmsf run build

# 2) bin 파일이 어떻게 동작하는지 확인 (--version만 출력하고 종료)
node packages/create-kmsf/bin/create-kmsf.js --version
# → 0.1.0 출력

# 3) dist/ 안 들여다보기
ls packages/create-kmsf/dist/
# args.js  banner.js  cli.js  logger.js  prompts.js

3.8 신입에게 흔한 함정

🚨 dist/ 가 없으면 ERR_MODULE_NOT_FOUND

방금 git clone 받았다면 dist/ 폴더가 없습니다 (gitignore). 그래서 첫 실행 전 반드시 npm run build를 워크스페이스별로 실행하세요.
