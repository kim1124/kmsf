Chapter 5 / 10

scaffold() 9단계

한 줄 요약

scaffold()는 정해진 순서로 9단계를 실행한다: 안전 점검 → 복사 → auth-mode 적용 → package.json 변환 → 토큰 치환 → .env.local 생성 → npm install → playwright → git init. 순서를 바꾸면 망가진다.

5.1 위치 + 역할

scaffold()는 packages/generator-core/src/index.ts에 있는 함수입니다. cli.ts에서 import해서 호출합니다. 이 함수는 오케스트레이터(지휘자) 역할 — 9개의 작은 모듈을 정해진 순서로 호출합니다.

5.2 9단계 전체 흐름

파란색 = 안전 점검 / 노란색 = 파일 복사 / 초록색 = 변환 / 분홍색 = post-install (선택적)

5.3 왜 이 순서여야 하나? (불변 규칙)

9단계의 순서는 임의로 바꾸면 안 되는 불변 규칙입니다 (도메인문서.md §3.1.1). 각 단계는 이전 단계의 결과를 가정합니다.

순서이유
① 템플릿 존재 확인 먼저없는 거 알면 다른 일 시작 전에 빨리 종료
② 비어있는지 확인 먼저이미 있는 폴더 덮어쓰면 사용자 작업 날아감
③ copyDir이 4보다 먼저복사가 안 되면 변환할 파일이 없음
④ applyAuthMode가 5보다 먼저제거된 파일에 변환 적용하면 헛수고
⑤ transformPackageJson이 6보다 먼저의존성 정리가 끝난 후에 토큰 치환
⑥ substituteTokens가 7보다 먼저토큰 치환 끝난 상태에서 .env.local 생성
⑦ .env.local이 8보다 먼저npm install이 환경 변수에 의존할 수 있음
⑧ install이 9보다 먼저node_modules 없으면 playwright/git 작업 의미 없음

🚨 절대 하지 말 것

이 순서를 "효율 개선"이라며 바꾸지 마세요. 예를 들어 "복사 전에 토큰 치환"을 시도하면 — 치환할 파일이 없어서 아무 일도 일어나지 않습니다. 또는 "auth-mode를 토큰 치환 후"로 바꾸면 — 이미 치환된 코드가 다시 제거되어 일관성이 깨집니다.

5.4 호출 계통도 (orchestrator 시점)

scaffold()는 자식 모듈 9개를 순서대로 호출하는 지휘자다.

5.5 진행 상황 출력 (logger)

각 단계에 들어가기 전에 logger.step()을 부르고, 끝나면 logger.stepDone()을 부릅니다. 사용자는 이런 모양으로 봅니다.

터미널 출력 예시 
› Copying template ...
✔ Copying template (45ms)
› Configuring auth: local-json ...
✔ Configuring auth: local-json (3ms)
› Configuring package.json ...
✔ Configuring package.json (1ms)
› Substituting tokens ...
✔ Substituting tokens (12ms)
› Generating .env.local ...
✔ Generating .env.local (1ms)

✨ Done! Created my-app at /tmp/kmsf-test/my-app

Next steps:
  cd my-app
  npm install
  npm run dev

5.6 post-install 실패는 throw가 아니라 warning

⑧⑨ 단계 (npm install, playwright, git init)는 실패해도 throw 안 합니다. { success: false, reason }을 받아 warnings[]에 누적하고, 마지막에 사용자에게 출력합니다. 이유: 부분 실패 시에도 사용자가 수동 복구할 수 있게.

scaffold() 내부 
if (options.runInstall) {
  const r = await runInstall(options.targetDir, options.packageManager);
  if (r.success) {
    options.logger.stepDone(r.durationMs);
  } else {
    options.logger.stepFailed(r.reason ?? "unknown");
    warnings.push(`install: ${r.reason}`);  // ← throw 안 함
  }
}
